📑 Техническое Задание: VPN Telegram Bot (VLESS Reality + ЮKassa)
1. Общее описание

Коммерческий Telegram-бот для автоматизации продажи и управления VPN-подписками.

    Протокол: VLESS Reality.

    Backend: Панель 3x-ui.

    Клиентское ПО: Hiddify (основной), v2rayNG, V2Box.

2. Стек технологий

    Language: Python 3.10+

    Bot Framework: aiogram 3.x (асинхронный)

    Database: SQLite (через aiosqlite)

    Payments: ЮKassa API

    Network: aiohttp для запросов к XUI и платежам.

3. Архитектура Базы Данных

Необходимо мигрировать с текущей таблицы trial_users на следующую структуру:
3.1. Таблица users
Поле	Тип	Описание
telegram_id	INTEGER (PK)	ID пользователя в Telegram
uuid	TEXT	VLESS UUID (ключ доступа)
email	TEXT	Уникальное имя в XUI (формат: tg_123456)
port	INTEGER	Порт подключения
expiry_time	INTEGER	Unix timestamp (ms) окончания подписки
ip_limit	INTEGER	Лимит устройств (IP)
short_id	TEXT	ID для короткой ссылки подписки
is_active	BOOLEAN	Статус (1 - работает, 0 - отключен)
warning_sent	BOOLEAN	Флаг уведомления об окончании (за 24ч)
3.2. Таблица payments
Поле	Тип	Описание
id	INTEGER (PK)	ID транзакции
payment_id	TEXT	ID в системе ЮKassa
telegram_id	INTEGER	ID плательщика
amount	INTEGER	Сумма платежа
status	TEXT	Статус (pending, succeeded, canceled)
tariff_data	TEXT	JSON с параметрами (дни, лимит IP)
4. Бизнес-логика и Тарифы
4.1. Сетка тарифов (config.py)

Тарифы формируются по матрице: Срок (1, 3, 6 мес) x Устройства (1, 3, 5 шт).

    Пример: "1 мес / 1 устр." — 150₽, "3 мес / 3 устр." — 400₽.

4.2. Алгоритм оплаты

    Пользователь выбирает тариф в меню.

    Бот создает платеж через ЮKassa API, сохраняет payment_id в БД.

    Бот выдает инвойс (ссылку) пользователю.

    При получении статуса succeeded:

        Если юзера нет в БД: Создаем новый Inbound в XUI + запись в БД.

        Если юзер есть: Продлеваем expiry_time (текущая дата + дни тарифа) и обновляем limitIp.

        Отправляем сообщение с ключом и инструкцией.

5. Взаимодействие с 3x-ui (xui_api.py)

Необходимые функции:

    add_client(email, uuid, limit_ip, expiry) — создание.

    update_client(email, limit_ip, expiry) — продление и изменение лимитов.

    enable_client(email) / disable_client(email) — управление доступом.

6. Пользовательский интерфейс (UI)
6.1. Главное меню
    Пробная подписка (сохрани существующий функционал)
    💎 Купить / Продлить — запуск сценария оплаты.

    👤 Мой профиль — инфо о подписке (статус, дата, лимит).

    📖 Как подключить — инструкции для Hiddify.

6.2. Инструкция (Hiddify)

Бот должен выдавать:

    Ссылку на скачивание (App Store / Google Play / GitHub).

    Кнопку "Скопировать ключ".

    Картинку-QR код для быстрого импорта.

7. Автоматизация (Background Tasks)

Реализовать асинхронный цикл (раз в 1 час):

    Поиск пользователей, у которых до конца подписки < 24 часов и warning_sent = 0. Отправка уведомления.

    Поиск просроченных подписок. Смена is_active = 0 в БД (XUI отключит их сам по дате).

8. Дорожная карта (Roadmap)

    Этап 1: Рефакторинг database.py и config.py под новую структуру.

    Этап 2: Реализация модуля yookassa_pay.py (создание и проверка платежей).

    Этап 3: Доработка xui_api.py (функция продления клиента).

    Этап 4: Создание интерфейса покупки в handlers.py.

    Этап 5: Написание планировщика (Scheduler) для уведомлений.

⚠️ Важные примечания для разработки

    Всегда использовать logging для отслеживания ошибок API.

    Хранить секретные ключи только в .env.

    При продлении подписки прибавлять время к expiry_time пользователя, а не к текущему моменту (чтобы дни не сгорали при раннем продлении).
